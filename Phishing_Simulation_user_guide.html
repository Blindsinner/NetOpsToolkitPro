üõë Why Spoofing Often Fails

Most modern email servers validate:

  • SPF: Ensures the sending IP is authorized for the domain
  • DKIM: Verifies message was not tampered with
  • DMARC: Dictates what to do when SPF/DKIM fails (reject/spam)
Spoofed emails using third-party SMTPs (like Hostinger, Gmail, Outlook) will often be rejected or land in spam due to failed SPF/DKIM.

‚úÖ How to Increase Deliverability of Spoofed Emails

  • Use your own SMTP server (Postfix/Exim) on a VPS
  • Control DNS to configure SPF/DKIM/DMARC
  • Send as your domain (or lookalike domains) to bypass SPF/DKIM blocks
  • Use internal test servers (MailSlurp, Mailhog) for simulations
  • Consider phishing platforms (GoPhish, King Phisher) for robust spoofing support

üîß Setup: Spoof-Ready SMTP Server on VPS

Requirements

  • Ubuntu VPS (DigitalOcean, Hetzner, Linode, etc.)
  • Domain registered (e.g., my-redteam-tools.com)
  • Root access to install mail tools

1. Install Postfix

sudo apt update && sudo apt install postfix mailutils
Choose "Internet Site" when prompted.

2. Configure Postfix to Allow Spoofing

Edit /etc/postfix/main.cf: 
myhostname = mail.my-redteam-tools.com
myorigin = /etc/mailname
inet_interfaces = all
inet_protocols = all
mydestination = $myhostname, localhost.$mydomain, localhost
smtpd_banner = $myhostname ESMTP
smtpd_recipient_restrictions = permit_mynetworks, permit_sasl_authenticated, reject_unauth_destination

3. Add Domain to DNS

Create DNS records:
  • SPF: v=spf1 mx ~all
  • DKIM: Optional (use opendkim if needed)
  • DMARC: v=DMARC1; p=none (allows spoofing)
  • Reverse DNS: Point your VPS IP to mail.my-redteam-tools.com

4. Restart Postfix

sudo systemctl restart postfix

5. Test with Your Tool

Use SMTP config:
  • SMTP Server: mail.my-redteam-tools.com
  • Port: 25 or 587 (with STARTTLS)
  • User: Any (if SMTP AUTH is enabled)
  • From: spoofed@target.com (Your tool allows this)
WARNING: Spoofing external domains without consent is illegal in most jurisdictions. Use only during authorized red team operations.
